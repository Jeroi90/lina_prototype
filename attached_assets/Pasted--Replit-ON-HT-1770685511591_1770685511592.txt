📋 Replit 개발 요청 프롬프트 (기획서)
[프로젝트 개요]

목표: 보험금 청구 및 상품 후기 공유 커뮤니티 '라이나ON'의 모바일 웹앱 프로토타입 구현

기술 스택: HTML5, Tailwind CSS (CDN), Vanilla JavaScript (SPA 구조)

현재 상태: sty-001.html (메인), sty-002.html (대상선택), sty-003.html (에디터), sty-004.html (상세), sty-005.html (마이페이지)로 UI가 분리되어 있음.

요청 사항: 위 파일들을 하나의 index.html (SPA 구조)로 통합하고, [후기 작성 및 피드 등록] 시나리오가 실제로 작동하도록 자바스크립트 로직을 구현해줘.

[구현 시나리오: 후기 작성 및 등록]

사용자가 메인 피드에서 시작하여, 후기를 작성하고 완료하면 다시 메인 피드에 내가 쓴 글이 노출되는 흐름입니다.

1. 메인 피드 (Screen: STY-001)

앱 실행 시 가장 먼저 보여지는 화면입니다.

자바스크립트 배열(feedData)에 저장된 더미 데이터들을 기반으로 피드 카드를 동적으로 렌더링해야 합니다.

우측 하단 **'이야기 쓰기' (Floating Button)**를 클릭하면 **[대상 선택 화면]**으로 전환(화면 이동 효과)됩니다.

2. 작성 대상 선택 (Screen: STY-002)

'보상 후기', '상품 후기', '자유 수다' 중 하나를 선택하는 화면입니다.

이번 시나리오에서는 '보상 후기' 탭의 리스트 중 하나(예: 치아보험)를 클릭하면 **[후기 에디터 화면]**으로 이동합니다.

3. 후기 작성 에디터 (Screen: STY-003 + STY-004 통합)

별점 주기, 태그 선택, 텍스트 입력 기능이 작동해야 합니다.

필수 조건: 별점, 태그, 본문 내용이 입력되어야 하단 '등록하기' 버튼이 활성화(Disabled 해제) 됩니다.

'등록하기' 버튼 클릭 시:

입력된 데이터(제목, 내용, 별점, 태그 등)를 객체로 생성합니다.

feedData 배열의 **맨 앞(0번 인덱스)**에 이 객체를 추가합니다.

"등록이 완료되었습니다" 알림창(alert) 후 **[메인 피드]**로 복귀합니다.

4. 결과 확인 (Screen: STY-001)

메인 피드로 돌아왔을 때, 방금 작성한 글이 피드 최상단에 노출되어야 합니다. (리렌더링 필수)

[개발 상세 가이드라인]

1. SPA(Single Page Application) 구조 적용

모든 화면(section 태그 등)을 index.html 하나에 넣고, CSS 클래스(hidden 또는 active)를 조작하여 화면 전환을 구현해주세요.

화면 전환 시 부드러운 애니메이션(Fade-in 등)을 적용해주세요.

2. 데이터 구조 (Data Structure)

전역 변수 let feedData = [...]를 선언하고 초기 더미 데이터를 5~6개 정도 넣어주세요.

데이터 객체 구조 예시:

JSON
{
  "id": 1,
  "type": "claim", // claim, product, lounge
  "title": "임플란트 치료 후기",
  "desc": "생각보다 안 아프고 입금도 빨랐어요.",
  "author": "김라이나",
  "date": "방금 전",
  "likes": 0,
  "badge": "가입 3년차",
  "star": 5
}
3. UI/UX 디테일

헤더 처리: 각 화면(메인, 작성, 선택)마다 헤더의 구성(뒤로가기 버튼 유무, 타이틀)이 다르므로 상황에 맞게 변경되거나 교체되어야 합니다.

입력 편의성: 에디터 화면에서 텍스트 영역을 클릭하면 시연을 위해 긴 글이 자동으로 입력되는 '오토 타이핑(Auto-typing)' 기능을 넣어주세요.

4. 코드 작성 요청

위 내용을 바탕으로 index.html (전체 구조 및 로직), style.css (필요시), script.js (화면 전환 및 데이터 처리 로직) 코드를 작성해줘.

특히 submitReview() 함수에서 데이터를 배열에 추가하고 메인 화면을 갱신하는 로직을 꼼꼼하게 짜줘.
